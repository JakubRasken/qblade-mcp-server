# QBlade MCP Server

An official Model Context Protocol (MCP) server for the **QBlade** Wind Turbine Simulation and Design software. This server equips LLM agents (like Claude, Gemini, or other agentic frameworks) with the power to compile QBlade, programmatically generate turbine configurations, execute wind turbine simulations (with headless auto-fallback), and parse output data logs for performance evaluation.

---

## Features & Tools

This MCP server exposes the following tools:

### ⚙️ `compile_qblade`
* **Purpose**: Automatically runs `qmake` and `make` against the local QBlade source directory to compile the `QBladeCE` (or `QBladeEXE`) executable.
* **Arguments**:
  * `forceClean` (boolean, optional): Performs a `make clean` before recompiling.
  * `threads` (string, optional): Number of compilation threads. Default is `"all"` (using `$(nproc)`).

### 🚀 `run_qblade`
* **Purpose**: Launches the compiled QBlade binary with a designated project/config input file.
* **Special Capabilities**:
  * **Headless Mode**: Standard graphical applications crash on remote servers or WSL instances without active displays. By default, this tool runs under a virtual framebuffer (`xvfb-run -a`) to support automation and scripting.
  * **Interactive/WSLg Support**: Set `headless` to `false` to launch it inside your host graphical environment (e.g. WSLg or native X server).
  * **Timeout**: Set a `timeoutMs` limit to automatically terminate QBlade after the simulation completes, allowing agentic loops to run uninterrupted.

### 📝 `generate_qblade_configs`
* **Purpose**: Programmatically generates valid text templates/configurations in QBlade's custom format.
* **Arguments**:
  * `type` (enum: `"simulation"`, `"turbine"`, `"wind"`, `"airfoil"`): Config file type.
  * `outputPath` (string): Absolute path to save the generated config.
  * `params` (object, optional): Configuration overrides (e.g. wind speeds, rotor diameters, pitch).

### 📊 `read_qblade_output`
* **Purpose**: Parses space/tab-separated simulation reports generated by QBlade.
* **AIAgent Friendly**: Instead of dumping thousands of raw timeseries rows that would saturate LLM context windows, it returns header names, sample rows, and computed statistics (min, max, mean, count) for every numeric column.

## Installation & Requirements

### 1. System Requirements
To build and run QBlade, ensure you have the required system compilation tools and libraries installed:
```bash
sudo apt-get update
sudo apt-get install -y build-essential qt5-qmake qtbase5-dev libgl1-mesa-dev libglu1-mesa-dev libqt5opengl5-dev libqt5xml5-dev libquadmath0 xvfb
```

### 2. Node.js Environment
Make sure Node.js (v18+) is installed. In this server's directory, run:
```bash
npm install
```

## Connecting to Claude Desktop

Add the server to your `claude_desktop_config.json` (typically located in `%APPDATA%/Claude/claude_desktop_config.json` on Windows or `~/.config/Claude/claude_desktop_config.json` on macOS/Linux):

```json
{
  "mcpServers": {
    "qblade-mcp-server": {
      "command": "node",
      "args": ["/home/jakub/qblade-mcp-server/index.js"]
    }
  }
}
```
